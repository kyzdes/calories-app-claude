## 1. Философия дизайна

### Принципы

| # | Принцип | Правило | Проверочный вопрос |
|---|---------|---------|-------------------|
| P1 | **Speed First** | Любое действие — максимум 3 тапа | «Можно ли убрать ещё один тап?» |
| P2 | **Less is More** | Один экран = одна задача | «Есть ли здесь что-то, без чего можно обойтись?» |
| P3 | **Clarity > Decoration** | Данные читаются за 1 секунду | «Пользователь понимает цифры без подсказки?» |
| P4 | **Zero Friction** | Ноль попапов, рекламы, upsell | «Этот элемент помогает или мешает задаче?» |
| P5 | **Honest Data** | Округление до целых, без ложной точности | «Мы не создаём иллюзию точности?» |

### Визуальный язык

- **Стиль:** Flat-дизайн в духе Apple HIG. Нет градиентов, теней, 3D-эффектов.
- **Ощущение:** Чисто, спокойно, уверенно. Приложение-инструмент, не приложение-развлечение.
- **Метафора:** Цифровой дневник — лаконичный, аккуратный, приятный в использовании.

## 3. Типографика

**Шрифт:** SF Pro (системный). Для крупных цифр калорий — SF Pro Rounded.

### Шкала типографики

| Token | Шрифт | Размер | Вес | Line Height | Использование |
|-------|-------|--------|-----|-------------|---------------|
| `type.hero` | SF Pro Rounded | 48pt | Bold | 52pt | Число калорий в круговом прогрессе |
| `type.h1` | SF Pro | 28pt | Bold | 34pt | Заголовки экранов |
| `type.h2` | SF Pro | 22pt | Semibold | 28pt | Заголовки секций |
| `type.h3` | SF Pro | 18pt | Semibold | 24pt | Заголовки карточек |
| `type.body` | SF Pro | 16pt | Regular | 22pt | Основной текст |
| `type.body.bold` | SF Pro | 16pt | Semibold | 22pt | Акцентный текст в body |
| `type.callout` | SF Pro | 15pt | Regular | 20pt | Подписи, вторичная информация |
| `type.caption` | SF Pro | 13pt | Regular | 18pt | Мелкие подписи, timestamps |
| `type.caption2` | SF Pro | 11pt | Regular | 14pt | Третичная информация |
| `type.number.lg` | SF Pro Rounded | 32pt | Bold | 36pt | Крупные числа (БЖУ суммы) |
| `type.number.md` | SF Pro Rounded | 20pt | Semibold | 24pt | Числа в карточках продуктов |
| `type.number.sm` | SF Pro | 14pt | Medium | 18pt | Граммовки, мелкие цифры |

### Правила типографики

- Калории всегда отображаются **без десятичных** — `1842 ккал`, не `1842.3 ккал`
- БЖУ отображаются с точностью до **1 г** — `124 г`, не `124.2 г`
- Вес — с точностью до **0.1 кг** — `78.3 кг`
- Единицы измерения всегда через неразрывный пробел: `124\u00A0г`, `1842\u00A0ккал`
- Для разделения тысяч используется тонкий пробел: `1\u2009842 ккал`

## 5. Сетка и отступы

### Spacing Scale

| Token | Значение | Использование |
|-------|----------|---------------|
| `space.2xs` | 2pt | Микроотступы внутри компонентов |
| `space.xs` | 4pt | Отступ между иконкой и текстом |
| `space.sm` | 8pt | Отступы внутри compact-компонентов |
| `space.md` | 12pt | Стандартный внутренний отступ |
| `space.base` | 16pt | Основной горизонтальный padding экрана |
| `space.lg` | 20pt | Отступ между секциями |
| `space.xl` | 24pt | Отступ между крупными блоками |
| `space.2xl` | 32pt | Отступ перед заголовками секций |
| `space.3xl` | 40pt | Крупные разрывы между разделами |
| `space.4xl` | 48pt | Верхний отступ контента от navigation bar |

### Layout Grid

- **Горизонтальный padding экрана:** 16pt (обе стороны)
- **Content width:** Полная ширина экрана минус 32pt
- **Карточки:** Полная ширина контента (edge-to-edge внутри padding)
- **Минимальная высота строки списка:** 44pt (тапабельная область)
- **Safe Area:** Всегда учитывается. Контент никогда не залезает под нотч/Dynamic Island

3. Open Food Facts API

### Debounce

- Задержка: 300ms после последнего символа
- Минимум символов: 2
- Отмена предыдущего запроса при новом вводе

## 16. Состояния и Edge Cases

### 16.1 Глобальные состояния

| Состояние | Поведение |
|-----------|-----------|
| Первый запуск | → Onboarding |

| Нет интернета | Приложение работает полностью. При поиске: «Только локальные результаты». Иконка offline в nav bar |
| HealthKit недоступен | Раздел скрыт в профиле. Приложение работает без него |

| Состояние | Визуал |
|-----------|--------|
| Нет недавних | Секция «Недавние» скрыта |
| Нет частых | Секция «Частые» скрыта |
| Нет ни недавних, ни частых | Только поиск + «Создать продукт» |
| Поиск: загрузка | Skeleton-loader (3 строки с пульсацией) |
| Поиск: нет результатов | Текст + кнопка «Создать продукт» |

## 17. Анимации и переходы

### 17.1 Таблица анимаций

| Элемент | Тип | Параметры | Длительность |
|---------|-----|-----------|-------------|
| Круговой прогресс: заполнение | Spring | `response: 0.6, dampingFraction: 0.8` | ~0.6s |
| Круговой прогресс: изменение числа | Numeric | `.contentTransition(.numericText())` | 0.3s |
| БЖУ-бары: заполнение | Spring | `response: 0.5, dampingFraction: 0.85` | ~0.5s |
| Meal card: появление продукта | Insert | `.asymmetric(insertion: .slide, removal: .opacity)` | 0.3s |
| Meal card: удаление продукта | Remove | `.opacity + .slide(edge: .trailing)` | 0.25s |
| Sheet: появление | System | iOS default sheet presentation | ~0.3s |
| Sheet: закрытие | System | iOS default sheet dismissal | ~0.3s |
| Day switch | Slide | `.slide(edge: direction)` | 0.25s |
| Search results | Fade | `.opacity` | 0.2s |
| Graph: столбцы | Spring staggered | `delay: index * 0.05`, spring | ~0.5s |
| Graph: period switch | Morph | `.interpolatingSpring` | 0.4s |
| Toast: появление | Move+Fade | `from bottom, opacity` | 0.3s |
| Toast: исчезновение | Fade | `.opacity` | 0.3s |
| Scan success | Scale+Fade | `scale(1.0 → 1.1 → 1.0)` | 0.2s |

### 17.2 Принципы анимации

- **Никаких декоративных анимаций.** Каждая анимация должна нести информационный смысл.
- **Spring > linear.** Используем spring-анимации как основные.
- **Reduce Motion.** Если `.accessibilityReduceMotion == true` → все анимации заменяются на `opacity` transitions.
- **Никаких задержек при запуске.** Splash screen → дашборд без лишних анимаций.

### Push-уведомления (опционально, запрос после 3 дней использования)

| Уведомление | Триггер | Текст | Время |
|-------------|---------|-------|-------|
| Напоминание о логировании | Нет записей после 12:00 | «Не забудьте записать обед 🍽» | 13:00 |
| Напоминание о логировании | Нет записей после 18:00 | «Как прошёл ужин? Запишите для отслеживания» | 20:00 |
| Напоминание о весе | Нет записи веса > 3 дней | «Давно не записывали вес. Как дела?» | 09:00 |
| Стрик | 7 дней подряд логирования | «7 дней подряд! Отличная привычка 🔥» | После последнего лога |

### Правила уведомлений

- Запрос разрешения: **не при первом запуске**. Через 3 дня использования или после 10 логирований.
- Максимум: 1 уведомление в день.
- Тон: поддерживающий, не навязчивый. Без guilt-tripping.
- Отключаемые: каждый тип отдельно в Профиле.

## 20. Accessibility

### VoiceOver

| Элемент | accessibilityLabel | accessibilityHint |
|---------|-------------------|------------------|
| Круговой прогресс | «Калории: съедено 1245 из 2100. Осталось 855» | — |
| БЖУ-бар (белки) | «Белки: 87 из 158 грамм» | — |
| Meal card | «Завтрак, 428 килокалорий, 2 продукта» | «Нажмите для подробностей» |
| Продукт в списке | «Овсянка с бананом, 100 грамм, 245 килокалорий» | «Нажмите для редактирования. Смахните влево для удаления» |
| FAB | «Добавить еду» | «Откроет экран добавления продукта» |
| Date switcher | «Сегодня, 21 февраля» | «Нажмите для выбора даты» |
| Кнопка сканера | «Сканировать штрих-код» | «Откроет камеру для сканирования» |
| Graph | «График калорий за неделю. Среднее 1856 килокалорий» | «Нажмите на точку для подробностей» |

### Dynamic Type

- Все тексты поддерживают Dynamic Type.
- Минимальный размер: не ограничиваем (система решает).
- При увеличении шрифта: карточки растягиваются вертикально, горизонтальные layout'ы становятся вертикальными (через `@ScaledMetric` и `ViewThatFits`).
- Графики: фиксированная высота, легенда масштабируется.

### Контрастность

- Все текстовые элементы: контрастность ≥ 4.5:1 (WCAG AA).
- Крупный текст (≥ 18pt bold): контрастность ≥ 3:1.
- Прогресс-бары: цвет заполнения vs фон ≥ 3:1.

### Reduce Motion

- Если включено: все spring-анимации → `opacity` transitions.
- Графики: без staggered animation, мгновенное появление.
- Свайпы переключения дней: мгновенный переход.

## 21. Dark Mode

### Принципы

- Dark mode поддерживается с первой версии.
- Используем `Color` из asset catalog с Light/Dark вариантами.
- Нет «серого» dark mode — используем true black (#0A0A0A) для OLED-экранов.
- Прогресс-бары: цвета насыщеннее в dark mode (см. таблицу токенов).

### Специфика

| Элемент | Light | Dark |
|---------|-------|------|
| Фон основной | #FFFFFF | #0A0A0A |
| Фон карточек | #F5F5F5 | #171717 |
| Фон инпутов | #EBEBEB | #262626 |
| Primary | #1B6B4A | #34D399 |
| Круговой прогресс (track) | #EBEBEB | #262626 |
| Графики (фон) | #F5F5F5 | #171717 |
| Tab bar | System blur (light) | System blur (dark) |
| FAB shadow | Visible | Hidden (не нужна на тёмном фоне) |
| Разделители | #E5E5E5 | #333333 |

### Изображения и иконки

- SF Symbols автоматически адаптируются.
- Кастомных изображений в MVP нет → нет проблемы.
- QR/barcode viewfinder: белая рамка на обоих темах.
